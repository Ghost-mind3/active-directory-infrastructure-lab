# 09 - Advanced Audit Policy (GPO)

## Overview

This step focuses on implementing an **Advanced Audit Policy** using Group Policy in order to improve visibility and monitoring within the Active Directory environment.

Audit policies allow administrators to **track authentication events, account changes, and privilege usage** across domain systems.  
These logs are essential for **security monitoring, incident response, and forensic analysis**.

The configuration was implemented using a dedicated **Group Policy Object (GPO)**.

---

## Objective

The goal of this configuration is to:

- Monitor authentication activity
- Detect failed login attempts
- Track account modifications
- Identify privilege abuse
- Improve domain security visibility
- Track Active Directory object changes

Audit logs generated by this policy can be analyzed through **Windows Event Viewer**.

---

# GPO Creation

The audit configuration was implemented through a new Group Policy Object.

### Steps

1. Open **Group Policy Management Console (GPMC)**
2. Navigate to the domain:

```
Forest: evilcorp.local
└── Domains
    └── evilcorp.local
```

3. Right-click **Group Policy Objects**
4. Select **New**
5. Name the policy:

```
GPO-Audit-Policy
```

---

# Editing the GPO

After creating the policy, it must be edited to configure the audit settings.

### Steps

1. Right-click **GPO-Audit-Policy**
2. Click **Edit**
3. Navigate to:

```
Computer Configuration
└── Policies
    └── Windows Settings
        └── Security Settings
            └── Advanced Audit Policy Configuration
                └── Audit Policies
```

---

# Account Logon Auditing

This category monitors **authentication activity within the domain**.

These events are primarily generated on **Domain Controllers**.

### Configuration Path

```
Audit Policies
└── Account Logon
```

### Configured Settings

The following audit events were enabled:

- Audit Credential Validation → **Success and Failure**
- Audit Kerberos Authentication Service → **Success and Failure**
- Audit Kerberos Service Ticket Operations → **Success and Failure**
- Audit Other Account Logon Events → **Success and Failure**

### Screenshot

![Account Logon Audit](Images/AuditGPO.png)

### Security Benefit

This configuration allows detection of:

- Failed authentication attempts
- Kerberos ticket abuse
- Credential validation attempts
- Potential brute-force attacks

# Directory Service Changes

This audit policy tracks **modifications made to Active Directory objects**.

It is particularly useful for monitoring changes to:

- Users
- Groups
- Organizational Units
- Security attributes

### Configuration Path

```
Audit Policies
└── DS Access
```

### Configured Setting

- Audit Directory Service Changes → **Success**
![DS-access](Images/DS-access.png)
### Security Benefit

This configuration records events when:

- Active Directory objects are modified
- Group memberships are changed
- User attributes are updated
- Critical directory settings are altered

This provides important visibility for **security investigations and forensic analysis**.

# Key Takeaways

- Advanced Audit Policies provide **deep visibility into Active Directory activity**.
- Monitoring authentication events helps detect **brute force and credential attacks**.
- Account management auditing tracks **changes to users and groups**.
- Privilege auditing helps identify **potential privilege escalation attempts**.
- Directory Service auditing allows administrators to **monitor critical Active Directory changes**.
- Centralized audit logging is a **critical component of enterprise security monitoring**.
